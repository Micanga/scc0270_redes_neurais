Exercício 1¶

Pesquise e descreva a funo de ativao softmax.

RESPOSTA:

A funo softmax uma funo de ativao do tipo sigmide utilizada para auxiliar o aprendizado de redes para classificao. A funo softmax, diferentemente da sigmide pura, consegue resolver problemas multiclasses, sendo esta a principal diferena entre as duas funes de ativao.

Para isso, a funo softmax consegue apresentar, para a classificao, a probabilidade de uma entrada pertencer a uma determinada classe.

A funo de ativao softmax pode ser definida por:

$$ \sigma(z)_j = \frac{e^{z_j}}{\sum^K_{k=1}e^{z_k}}, para j = 1,2,...,K$$

com $z$ sendo a entrada e $K$ sendo a dimenso do vetor de entrada.

A funo softmax idealmente usada na camada de sada do classificador, atuando de maneira parecida com um normalizador do aprendizado/pesos da rede, na tentativa de gerar as probabilidades para definir a classe de cada entrada.

Exercício 2¶

Pesquise e descreva a funo de ativao ReLu.

RESPOSTA:

A ativao linear retificada (ReLU) e sua derivada so dadas, respectivamente, por:

$$ ReLU(x)=max\{0,x\}$$


$$ ReLU'(x)= \begin{cases} 1, & \text{se } x\ge 0\\ 0, & \text{c.c.} \end{cases}$$

A funo de ativao ReLU representa uma funo similar funo identidade, apresentando fcil otimizao e implementao. A principal e nica diferena entre a funo ReLU e a identidade que metade do domnio da funo ReLU representada por zeros. Como consequncia direta deste fato, as derivadas iro ser significantes somente e enquanto a unidade (neurnio) estiver ativa. Visualmente, o comportamento da funo ReLu, segue:

A principal vantagem associada s funes de ativao retificadas que sua no lineariedade tm oferecido a capacidade de se treinar de redes neurais profundas supervisionadas sem a necessidade de um pr-treino no supervisionado. Unidades de retificao linear permitem um treinamento mais veloz e efetivo de arquiteturas de redes neurais profundas sobre complexos e grandes conjuntos de dados.

Uma desvantagem da ativao ReLU que unidades tendem a morrer durante o treinamento, um fenmeno que faz com que o neurnio passe a produzir apenas zeros. Isso acontece quando a soma ponderada antes da aplicao da ReLU se torna negativa, fazendo com que a unidade produza zero. Nessa regio, a derivada tambm zero, fazendo com que os parmetros $w$ da unidade deixem de ser atualizados com gradiente descendente.

from mnist import MNIST
def load_mnist():
    mndata = MNIST('mnist_dataset')

    X_train, y_train = mndata.load_training()
    X_test, y_test = mndata.load_testing()

    X, y = [], []
    for i in range(len(X_train)):
        X.append(X_train[i])
        y.append(y_train[i])
    for i in range(len(X_test)):
        X.append(X_test[i])
        y.append(y_test[i])

    return X, y

def init_cnn(n_layers, num_classes, filter_sizes, alpha,\
input_shape=(28,28,1), kernel_size=(3,3), pool_size=(2,2), dropout=False, dropout_rate=0.3):
    model = keras.Sequential()

    # Input Layer
    model.add(keras.layers.Conv2D(filters=filter_sizes[0], kernel_size=kernel_size,activation='linear',\
                                input_shape=input_shape,padding='same'))
    model.add(keras.layers.LeakyReLU(alpha=alpha))
    model.add(keras.layers.MaxPooling2D(pool_size,padding='same'))
    if dropout:       
        if isinstance(dropout_rate, list):
            model.add(keras.layers.Dropout(dropout_rate[0]))
        else:
            model.add(keras.layers.Dropout(dropout_rate))

    # Hidden Layers
    for i in range(1,n_layers-1):
        model.add(keras.layers.Conv2D(filter_sizes[i], kernel_size, activation='linear',padding='same'))
        model.add(keras.layers.LeakyReLU(alpha=alpha))
        model.add(keras.layers.MaxPooling2D(pool_size,padding='same'))
        if dropout:           
            if isinstance(dropout_rate, list):
                model.add(keras.layers.Dropout(dropout_rate[i]))
            else:
                model.add(keras.layers.Dropout(dropout_rate))

    # Output Layer
    model.add(keras.layers.Flatten())
    model.add(keras.layers.Dense(filter_sizes[-1], activation='linear'))
    model.add(keras.layers.LeakyReLU(alpha=alpha))          
    if dropout:           
        if isinstance(dropout_rate, list):
            model.add(keras.layers.Dropout(dropout_rate[-1]))
        else:
            model.add(keras.layers.Dropout(dropout_rate))
    model.add(keras.layers.Dense(num_classes, activation='softmax'))

    # Compiling
    model.compile(loss=keras.losses.categorical_crossentropy,\
     optimizer=keras.optimizers.Adam(),metrics=['accuracy'])

    # Returning
    return model

cnn = init_cnn(4, 10, [32,64,128,128], 0.1,\
    input_shape=(28,28,1), kernel_size=(3,3), pool_size=(2,2),\
     dropout=True,dropout_rate=[0.25,0.25,0.4,0.3])
